🧪 MODULE 1 — Satellite Data as Numbers
A satellite image is NOT an image.
It is:
a matrix of numbers
each number = reflected energy from Earth surface
each band = a different wavelength measurement
So mentally convert this:
“image” ❌
 “multidimensional numerical dataset” ✅

1.  Understand GeoTIFF structure (Load GeoTIFF in Python)
Shape: {(Bands(no of bands), Height(pixel), Width(pixel)}
Example: (4, 10980, 10980)

2. Reflectance scaling problem
GEE often stores Sentinel-2 as: 0 → 10000
But real reflectance should be: 0 → 1
So conversion:
✅ Formula:
scaled_value = raw_value / 10000
If you skip this:
NDVI becomes inflated or wrong
comparison with GEE fails
ML features become meaningless
3. Compute NDVI manually
ndvi = (nir - red) / (nir + red)
This can be incorrect.

Handle real-world issues
Problem : division by zero and NaN pixel
ndvi = np.where((nir + red) == 0, np.nan, (nir - red) / (nir + red))
For each pixel:
If NIR + Red = 0 → set NDVI = NaN (invalid pixel)
Else → compute normal NDVI
Why this is needed
Prevents division by zero (0/0)
Keeps dataset scientifically valid
Marks missing/invalid pixels instead of creating errors
